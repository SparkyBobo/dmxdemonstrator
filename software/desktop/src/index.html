Overview

The DMX Demonstrator is a set of Arduino-based boards to create a DMX512-like link between and transmitter and receiver. The transmitter simulates a console with control of 4 channels, while the receiver simulates a fixture with 4 controllable parameters. The purpose of the DMX Demonstrator is to explain how the DMX512 protocol works by displaying the frame and data bits in real time.

What's the difference?

The DMX Demonstrator will let you step through a frame of a DMX-ish serial protocol. The primary differences between DMX512 and the DMX-Demonstrator are:
DMX512 DMX Demonstrator
Operates at 250k bits per second (bps) Operates from 1-200 and 3200 bits per second (bps). It can also be triggered to send the next bit manually, i.e., step mode
Operates asynchronously, meaning both sender and receiver agree on the data rate Operates synchronously to allow data rate to vary
Sends 512 data values per frame Sends 4 data values per frame
Signals are balanced, designed for multiple receivers, and perform well in noisy environments Signals are unbalanced, not designed for long distances or multiple devices
However, DMX512 and the DMX-Demonstrator share much in common:
DMX512 DMX Demonstrator
Operates within a specific set of timing parameters, such as the length of a break, and the minimum time needed to signal the start and end of data. Operates within the specific set of timing parameters relative to the data rate
Consists of multiple frame components: break, mark after break, start code start bit, start code, start code mark, value 1 start bit, etc... Consists of the same set of components as a DMX512 frame
Specified to work with a start code value of 0 Works with a start code value of 0, but this value can be changed by the sender or receiver
Designed to operate error-free Designed such that errors can be introduced on purpose to observe the effect

System Components

The required components for a DMX Demonstrator demo system are:
  • A transmitter: a DMX-TX1, a self contained transmitter
  • A receiver: a DMX-RX1, a self contained receiver with 4 dimmable LEDs
  • 2 compatible Arduino boards, such as Arduino Uno or Arduino Leonardo, or a clone of one of these boards
The optional components for a demo system include:
  • An alternate transmitter: a DMX-TX2 which uses a set of external dimmer and clock controls for more advanced operation.
  • A frame scrambler: the DMX-FS1, a transceiver with sets between the transmitter and receiver and can alter zero or more data bits. The frame scramble requires an additional Arduino board
  • An analog output module: a board that connects to the DMX-RX1 and provides analog voltage or pulse width modulated (PWM) signals
  • A digital output module: a board that connects to the DMX-RX1 and provides a DMX512 stream containing the 4 dimmer values received by the DMX-RX1. This module can also be connected to a DMX-TX1/2 to transmit the 4 dimmer values via DMX512
  • A constant voltage LED driver: a board that connects to the DMX-RX1 and provides a constant voltage pulse width modulated (PWM) signal for driving LEDs such as LED tape. This module can also be used to display the clock and data signals.
  • A constant current LED driver: a board that connects to the DMX-RX1 and provides a constant current pulse width modulated (PWM) signal for driving LEDs such as LED chips/modules. This module can also be used to display the clock and data signals.

Get Started

So let's get started! Grab a DMX-TX1, DMX-RX1, and 2 compatible Arduino boards.
  • Attach the DMX-TX1 to one of the Arduino boards
  • Attach the DMX-TX2 to the other Arduino board
  • Plug both Arduino boards into this computer
